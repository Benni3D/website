bcc - brainlet C compiler

The compiler is a small compiler for a C-like language. The language currently has a minimal type-system. (SEE LANGUAGE)

The compiler aims to implement a usable sub-set of the C99 standard, while changing some annoyance and adding a few extensions. Some of the changes break compatibility with the standard, this is intentional. The compiler does not support the full feature-set of neither C99 nor C90. But that was never the goal. This compiler was created as a learning project and as a successor to benc (an even worse compiler than this only available for 32-bit x86). Supported optimization techniques: - constant folding (-O1) - micro-optimizations (-O1) - multiply to shift (-O1) - fusion of memory instructions for x86 (-O2) - dead-code elimination (-O1) - constant evaluation of function call targets (-O1)

Currently supported target architectures: - i386 - x86_64

The compiler uses the System V ABI, where applicable. There is no multilib support, the compiler is installed for a single target-architecture. If a second target is wanted, the compiler must be rebuild and reinstalled for the specific architecure.
